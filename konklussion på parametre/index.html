Konklussion på parametre:

Det er den mest enkle måde at gemme og hente data på.

der er tre måder det kan bruges på: let, var, const

Variabler kan indeholde 7 forskellige værider, men kun bruges en ad gangen: Boolean Number String Object Null Undefined Symbol

TJEK NOTER HVIS DU VIL LÆSE MERE!!!

Kort sagt, så er Parametre værktøjer til at give funktioner data. Du skal være opmærksom på deres rækkefølge, antal og hvordan de kan påvirke ting udenfor funktionen.

Parametre som variabler eller faste værdier: Når vi bruger parametre, kan de enten være variabler, strings eller andre faste værdier. De fungerer som input til en funktion og bestemmer, hvordan funktionen udfører sit arbejde.

Hvordan parametre opfører sig: Parametre opfører sig som midlertidige variabler inden i funktionen. De får deres værdier fra de argumenter, vi sender med, når vi kalder funktionen.

Navne på parametre: Når vi definerer en funktion, giver vi parametrene navne. Disse navne fungerer som pladsholdere for de værdier, der bliver sendt med under et funktionskald. For eksempel:

function hej(navn) {

console.log("Hej, " + navn);

} hej("Anna"); // Her er "Anna" værdien for parameteren "navn".

Rækkefølgen af parametre: Rækkefølgen betyder noget! Når du kalder en funktion, skal værdierne (argumenterne) matche rækkefølgen af parametrene. Ellers kan funktionen få forkerte data.

Antallet af parametre: Hvis du sender flere eller færre argumenter, end funktionen forventer, kan det give problemer.

For få argumenter: Parametre uden værdi bliver undefined. For mange argumenter: De ekstra argumenter bliver ignoreret, medmindre funktionen er designet til at håndtere dem. Parametre, der påvirker andre variabler: Hvis parametrene ændrer værdier udenfor funktionen, kan det skabe uforudsigelig opførsel. Dette sker især, hvis du arbejder med objekter eller arrays, da de bliver refereret direkte i stedet for kopieret.
